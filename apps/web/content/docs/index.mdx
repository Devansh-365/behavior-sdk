# nyasa

`@devanshhq/nyasa` is a browser-side SDK that embeds into any web form and analyzes session behavior in real time. By the time a user hits submit, the SDK has already built a scored payload that tells your backend who filled the form.

## Why this exists

Bots and LLM agents have caught up with humans at the surface layer. A scripted bot can type, click, and paste. An LLM agent fills forms faster than most people but otherwise looks unremarkable. Traditional CAPTCHA and rate limiting catch neither reliably.

nyasa works at a different layer. It captures timing, movement, and environment signals that are much harder to fake consistently: the micro-variance in how long someone holds a key, whether the cursor follows a path that reflects human neuromuscular noise, whether the browser environment matches what a real device reports.

## The actor model

nyasa classifies every session into one of three categories:

| Actor | What it means |
|-------|--------------|
| **Human** | Organic behavioral signals and a consistent device fingerprint. No detection rules fired. |
| **Authorized agent** | A programmatic client with a valid cryptographic identity (RFC 9421). Not a threat. Flagged separately so your system can allow it intentionally. |
| **Unauthorized bot** | One or more detection rules fired. Could be a headless browser, a scripted automation, or an LLM agent acting without permission. |

## What the SDK does

- Attaches 12 behavioral collectors to a form element: keystroke, mouse, touch, paste, scroll, corrections, click precision, session rhythm, field timing, input origin, file upload, and tab visibility.
- Runs 8 browser environment checks: webdriver markers, WebGL renderer, canvas fingerprint, audio fingerprint, iframe consistency, incognito mode, timezone/locale consistency, and device persistence.
- Captures 3 network signals: focus-to-input reaction time, connection type, and page load timing.
- Runs detection rules client-side when the form submits.
- Sends the `BehaviorPayload` to your endpoint via `navigator.sendBeacon`, which is non-blocking and survives page unload.

## What the SDK does not do

nyasa is client-side only. It collects signals and runs pre-classification rules in the browser. It does not make a final fraud verdict. That is the job of the Zoven scoring API, which ingests the payload, runs the ML classifier, and returns an actor type and risk score to your fraud queue.
